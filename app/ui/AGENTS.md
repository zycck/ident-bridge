# Правила для `app/ui`

1. Интерфейс по умолчанию всегда светлый. Тёмные поверхности без явного выбора темы запрещены.
2. Любой `QMenu`, `QDialog`, popup списка или отдельное окно обязаны проходить через общий helper светлой палитры.
3. Нельзя писать raw hex-цвета прямо во view-модулях, кроме `theme`-слоя и одного центрального formatting-модуля, если без него нельзя.
4. Нельзя плодить однотипные helper-функции с разными именами. Один смысл — одна функция.
5. Нельзя дробить orchestration на микрофункции без прямой пользы для читаемости и тестирования.
6. Любой вызов к Qt popup/window styling, `QMenu`, `QDialog`, `QComboBox.view()` проверять глазами на Windows после правок.
7. Любой перенос модулей внутри `app/ui` делать через feature-пакеты и сохранять совместимость старых импортов.
8. Каноническая структура `app/ui`: `shared`, `dialogs`, `settings`, `sql_editor`, `export_jobs`, `dashboard`, `main_window`.
9. `threading.py` считается legacy-совместимостью. Новый код пишет только в `app.ui.shared.worker_threads`.
10. Перед завершением UI-задачи обязательно проверить:
   - нет битой кириллицы;
   - нет чёрных popup/menu/dialog;
   - тесты зелёные;
   - ключевые экраны открыты и просмотрены глазами.
